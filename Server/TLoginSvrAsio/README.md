# TLoginSvrAsio — modernized 4Story login server

Parallel reimplementation of `Server/TLoginSvr/` on top of a portable
C++20 stack: Boost.Asio coroutines, SOCI database access, OpenSSL
crypto, spdlog, toml++. Runs as a separate binary; the legacy
`TLoginSvr.exe` is untouched and can be A/B-compared during the
cutover. Wire-format byte-for-byte compatible with the shipped legacy
client (RC4 + XOR codec, RFC 6229 verified).

## Ported wire handlers

12 client→server handlers from the legacy `CSHandler.cpp` dispatcher:

| Handler | Status | Backed by |
|---|---|---|
| `CS_LOGIN_REQ` | ✅ real | `IAuthService::Authenticate` |
| `CS_GROUPLIST_REQ` | ✅ real | `IMapServerLocator::ListGroups` |
| `CS_CHANNELLIST_REQ` | ✅ real | `IMapServerLocator::ListChannels` |
| `CS_CHARLIST_REQ` | ✅ real | `ICharService::List` |
| `CS_CREATECHAR_REQ` | ✅ real | `ICharService::Create` |
| `CS_DELCHAR_REQ` | ✅ real | `ICharService::Delete` |
| `CS_START_REQ` | ✅ real | `IMapServerLocator::Lookup` (BR/BOW aware) |
| `CS_AGREEMENT_REQ` | ✅ real | `IAuthService::SetAgreement` |
| `CS_VETERAN_REQ` | ✅ real | `ICharService::GetVeteranLevels` |
| `CS_TERMINATE_REQ` | ⚠️ partial | closes session; full TLogoutAll deferred |
| `CS_HOTSEND_REQ` | 🚫 stub (legacy feature intentionally disabled) | — |
| `CS_SECURITYCONFIRM_ACK` | 🚫 stub (legacy dead code path) | — |

The 5 control-server handlers (`CT_SERVICEMONITOR_ACK`,
`CT_SERVICEDATACLEAR_ACK`, `CT_CTRLSVR_REQ`, `CT_EVENTUPDATE_REQ`,
`CT_EVENTMSG_REQ`) are not yet ported — they pair with `TControlSvr`,
which lands in Phase B.6.

## Service interfaces

Each service has an in-memory backend (zero-dep, used by ctest) and a
SOCI backend (DB-backed, used in production). Switching is a TOML
config change.

| Interface | In-memory | SOCI | Tests |
|---|---|---|---|
| `IAuthService` | ✅ | ✅ | 6 in-memory + 26 SOCI (PG + MSSQL) |
| `ICharService` | ✅ | ✅ | 6 in-memory + 39 SOCI |
| `IMapServerLocator` | ✅ | ✅ | 4 in-memory + 22 SOCI |
| `IConnectionRegistry` | ✅ | n/a (in-process only) | covered by duplicate-kick test |
| `ISessionTerminator` | ✅ | ✅ | 4 in-memory + 7 SOCI |

## Schemas

* `schema/postgres-dev.sql` — 20 legacy MSSQL tables converted to
  PG dialect (`SMALLINT` for `tinyint`, `TIMESTAMP` for
  `smalldatetime`, identity columns as `GENERATED BY DEFAULT AS
  IDENTITY`). Used by the integration test suite + dev binary.
* `schema/mssql-dev.sql` — 5 auth tables in original legacy MSSQL
  DDL (`tinyint`, `smalldatetime`, `int IDENTITY`). Applied to a
  containerized SQL Server 2022 to verify the dialect branches in
  the SOCI services work against the prod target.

## Configuration

Single TOML file. Empty `[database]` keeps the binary in
in-memory / dev mode — no DB needed for smoke tests. Setting
`backend` + `connection_string` is the one change required for
prod cutover.

```toml
[server]
port = 4815

[health]
port = 8815

[database]
# backend = "postgresql"     # postgresql | sqlite3 | odbc
# connection_string = "host=db.prod port=5432 dbname=tloginsvr user=svc password=..."
# pool_size = 32
```

Full schema with all keys: `tloginsvr.example.toml`.

## Build

CMake + vcpkg + system SOCI/spdlog/toml++:

```
$ cmake -B build -S .
$ cmake --build build --target tloginsvr_asio
$ build/bin/tloginsvr_asio --config tloginsvr.toml
```

Distro packages on Debian/Ubuntu cover everything:

```
$ apt install libsoci-dev libpq-dev unixodbc-dev libspdlog-dev \
              libtomlplusplus-dev libssl-dev libboost-all-dev
```

## What still needs work

**Phase B.5 (security)** — replace plaintext password compare in
`SociAuthService::VerifyPassword` with BCrypt + a transparent-upgrade
path that rewrites legacy plaintext rows with fresh `$2b$` hashes on
successful login.

**Phase B.6 (admin)** — the 5 `CT_*` control-server handlers plus the
UDP audit log shim that talks to `UdpSocket::LogLogin` in legacy. The
modern replacement should pipe these through `spdlog` to a structured
sink (Loki, Seq, …).

**Phase C (production validation)** — running against the real
production DB + real client. Items only verifiable there:

* legacy data shape quirks (encodings, NULL nuances, varchar lengths)
* real query plans / index tuning
* per-character map routing (legacy `CSPFindServerID` SP) — needs
  prod's map-id ↔ server-id table data we don't have in this repo
* tuned starter-item catalog (the current per-class placeholder set
  in `SociCharService` is scaffolding, not gameplay-final)

**Phase D (sibling servers)** — Patch, Log, Control modernization
(small, isolated). World/Map are massive (95k LOC) and only worth
modernizing for a concrete reason (cross-platform target, security
remediation, or vendor pressure).

See `_rewrite/docs/MODERNIZATION_PLAN.md` for the full roadmap.
